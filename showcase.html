System 01 of 04 · Live in Production

Manufacturing Operations Platform

Android shop floor digitisation · Live ERP integration · Brother label printing

AndroidPythonFlask SQL ServerMIE TrakREST API Brother SDKWindows Server
The Problem
  • Paper job packs assembled and distributed manually at each shift start.
  • Clock-in and clock-out recorded on paper, then entered into MIE Trak by hand after the shift.
  • No digital audit trail — no record of who worked on what, for how long, or when.
  • Handwritten move tickets: inconsistent, slow and error-prone.
  • End-of-shift data entry created daily admin overhead and ERP data lag.
Engineering Challenge
  • Replace paper job packs with digital equivalents on shop floor terminals.
  • Capture operator clock-in/out via Android, updating MIE Trak in real time.
  • Establish a full production audit trail — operator, job, work centre, timestamps.
  • Integrate the Brother label printer SDK for standardised move tickets.
  • Eliminate end-of-shift manual ERP data entry entirely.
The Solution
  • Android tablets at each work centre give operators direct access to their jobs, work instructions and tasks — no paper, no separate PC.
  • A Flask integration layer on the factory server connects every operator action straight to MIE Trak — no manual step between floor and ERP.
  • Live work order and job data drawn directly from the ERP database — current, not cached.
  • Standardised move tickets printed on demand at the point of production.
  • Runs on the factory LAN with no external internet dependency.

ERP Integration

Every operator action — clock-on, clock-off, job selection and quantity sign-off — updates the ERP at the moment it occurs. There is no end-of-shift upload, no manual data entry step, and no lag between what happens on the shop floor and what management sees. MIE Trak reflects the live state of production throughout the working day.

System 02 of 04 · Live in Production

Manufacturing Intelligence Platform

Real-time production dashboards · Executive & supervisor visibility · SQL-driven reporting

PythonFlaskSQL Server Jinja2MIE TrakWindows Server
The Problem
  • Management had no real-time view of the shop floor — status was known only at end-of-shift or after a floor walk.
  • Supervisors walked the floor or called operators to establish job progress — time lost from production management.
  • Overdue work orders were only visible with specialist ERP access — issues went unaddressed for hours.
  • Labour utilisation and operator efficiency were unknown during the working day.
Four Dashboard Views
  • Main Dashboard — live production summary; open and overdue work orders at a glance.
  • Live Shop Floor — every active operator session with elapsed time, job and work centre.
  • Employee Efficiency — actual vs. estimated time per operator; session-level performance.
  • Production Insights — work centre load, job distribution and production patterns.
The Solution
  • A dashboard server aggregates live production data and presents it in a browser — any device on the factory network, no login required.
  • All data drawn live from the MIE Trak database — the same source of truth that drives production.
  • Operator data from the Operations Platform feeds dashboards directly — shop floor and management share the same data, no delay.
  • No additional integration cost beyond the shared infrastructure already in place.

System 03 of 04 · Live in Production

Digital Job Pack Platform

Excel input · SQL lookup · PDF extraction · Consolidated print-ready job pack

PythonSQL ServerPDF Automation MIE TrakExcel / openpyxl
The Problem & Engineering Challenge

Job pack preparation was a time-intensive manual process: engineers or planners received a list of part numbers, then manually located the corresponding assembly PDFs across multiple customer folder locations, extracted the relevant drawings, and assembled them into a single job pack for the shop floor.

Repeated multiple times per day, it consumed significant engineering time, produced inconsistent results, and was a recurring source of production delays when documentation was missed, misfiled or assembled incorrectly. Preparation ranged from 30 minutes to several hours per batch.

The Automated Solution
  • Engineer uploads an Excel file of part numbers — the only manual step in the entire process.
  • Each part number matched automatically against the live ERP database — no manual searching across folder structures.
  • Relevant part drawings extracted from source PDFs automatically — only the required documentation included.
  • A single consolidated, correctly ordered, print-ready job pack PDF assembled for distribution.
  • Total preparation time: seconds per batch — no manual retrieval, sorting or assembly.

Measured Outcome
  • Job pack preparation reduced from hours to seconds — engineering time recovered for value-adding production work.
  • Production delays from missing, incorrect or out-of-order documentation significantly reduced.
  • Connected to live ERP data — the correct, current documents retrieved automatically, not a cached copy.
  • A repetitive daily task now handled automatically as the standard workflow.
Programme Notes
  • Connects to the live MIE Trak SQL Server database on every run — output based on current records, not a static cache.
  • Designed around the document types and folder structures already in use in production.
  • Run on demand by engineering or planning — no scheduled process or background service required.
  • Deployed on factory infrastructure without external development resource.

System 04 of 04 · Live in Production

Document Intelligence Platform

Drag-and-drop intake · Automatic part number detection · MIE Trak ERP filing and attachment

PythonMIE Trak APIPDF STEP FilesDXFERP Automation
The Problem & Engineering Challenge

Engineering documentation — PDFs, STEP files, DXF files — was filed manually into MIE Trak. Engineers received documents, identified the part number, navigated to the correct customer and part folder, placed the file, then manually attached it to the corresponding part record in the ERP.

This consumed engineering time on administration, and was a persistent source of downstream problems — documents filed in the wrong location, or not attached to the correct ERP record, creating delays across production, purchasing and quality.

The Solution — How It Works
  • Engineer drops documents onto the application — PDF, STEP and DXF all supported; no further manual input.
  • Part numbers identified automatically from document metadata and file naming — no manual identification, no misfiling.
  • Documents organised into the correct customer and part folder structure automatically.
  • Each file attached to the corresponding part record in MIE Trak automatically.
  • Document control becomes a consistent, automated process — correct practice enforced regardless of who handles the files.

  • Manual document filing substantially reduced — engineering time redirected to production-facing work.
  • Consistent, correctly structured document management enforced automatically — misfiling significantly reduced.

  • Document control managed through a system-enforced process rather than manual discipline.
